# API Design Standard
---
category: software-engineering
version: 1.0.0
---

## Objective
To ensure generated APIs are RESTful, consistent, and well-documented.

## Guidance
*   Use plural nouns for resource paths (e.g., `/users`, `/orders`).
*   Include standard HTTP status codes (200, 201, 400, 404, 500).
*   Structure JSON responses with a consistent "data" or "error" wrapper.
*   Ensure all endpoints are idempotent where appropriate.

## Prompt Standard
"When designing or implementing API endpoints, adhere to the ENGAGEntic API Design Standard by using RESTful resources and consistent JSON structures."
